🔥 טיפ Claude Code השבועי — CLAUDE.md
CLAUDE.md הוא ה-"מדריך הפרויקט לסוכן" — קובץ מרקדאון שקלוד קורא בתחילת כל session. בלי זה, קלוד שוכח הוראות כל פעם מחדש. עם זה — הוא תמיד יודע מה מותר, מה אסור, ואיך הפרויקט עובד.

# CLAUDE.md דוגמה בסיסית

## סביבת הפרויקט
- Node.js 20, TypeScript strict, pnpm
- Testing: Vitest | CI: GitHub Actions

## כללי קוד
- אל תשתמש ב-any — רק unknown
- כל PR דרך branch, לא ישירות ל-main

## מה לא לגעת
- legacy/old-api.ts — קוד ישן, לא לשנות
- .env.production — לעולם לא לקרוא

📌 עיקרון: settings.json = מה קלוד מורשה לעשות. CLAUDE.md = מה קלוד צריך לדעת. השתמש בשניהם!

⚡ טיפ טכני — Hooks לאוטומציה
Hooks ב-Claude Code = פקודות shell שרצות אוטומטית לפני/אחרי כל פעולה. הדרך הבטוחה לאכוף כללים שקלוד לא ישכח:

// .claude/settings.json
{
"hooks": {
"PostToolUse": [{
"matcher": "Write|Edit",
"command": "npx prettier --write $FILE"
}],
"PreToolUse": [{
"matcher": "Bash",
"command": "./scripts/block-dangerous-cmds.sh"
}]
}
}
👉 Docs רשמי: code.claude.com/docs
