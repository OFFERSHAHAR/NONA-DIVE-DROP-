KEY ALGORITHMS & FORMULAS
================================================================================

HAVERSINE DISTANCE (in useTrackingMap.ts):
const R = 6371000; // Earth radius in meters
const dLat = toRad(lat2 - lat1);
const dLon = toRad(lon2 - lon1);
const a = sin²(dLat/2) + cos(lat1) * cos(lat2) * sin²(dLon/2);
const c = 2 * atan2(√a, √(1-a));
const distance = R * c;

Returns: Distance in meters (accurate globally)

ETA CALCULATION:
eta_minutes = (distance_meters / (speed_ms * 3.6)) / 60
Default speed: 15 km/h if not provided
Result: Estimated minutes until arrival

NOTIFICATION TRIGGERS:
- distance < 50m → "Driver Arrived!"
- distance < 500m → "Driver Nearby"
- eta <= 1 minute → "1 Minute Away"
- eta <= 5 minutes → "5 Minutes Away"

All triggers use "already shown" flag to prevent duplicates.

================================================================================
ARCHITECTURE OVERVIEW
================================================================================

PASSENGER FLOW:
  Browser geolocation → useTrackingMap hook
    ↓
  Subscribe to shuttle_location_history changes
    ↓
  Calculate distance & ETA on each update
    ↓
  Check notification triggers
    ↓
  Update map markers & info card
    ↓
  Show notifications when thresholds hit

DRIVER FLOW:
  Device geolocation.watchPosition
    ↓
  Queue location every 15 seconds
    ↓
  POST /api/tracking/location
    ↓
  INSERT shuttle_location_history
    ↓
  UPDATE shuttle_trips current location
    ↓
  Realtime broadcast to passengers
    ↓
  Passengers receive update instantly

DATABASE FLOW:
  shuttle_location_history INSERT
    ↓
  Trigger: update shuttle_trips
    ↓
  Realtime replication active
    ↓
  Broadcast to all subscribed clients
    ↓
  Passengers receive in < 1 second
