================================================================================
                        ARCHITECTURE DECISIONS
================================================================================

1. SERVER ACTIONS vs API ROUTES STRATEGY

   Server Actions (for simple CRUD):
   - Direct database access
   - No JSON serialization overhead
   - Built-in CSRF protection
   - Simpler error handling
   - Location: src/lib/admin/server-actions.ts

   API Routes (for complex operations):
   - Batch operations with validation
   - File uploads (multipart/form-data)
   - External integrations
   - Mobile client support
   - Explicit HTTP methods
   - Location: src/app/api/admin/*

2. VALIDATION STRATEGY

   - Input: Zod schemas validate all inputs
   - Database: Constraints on schema
   - Output: Type-safe responses
   - Error Messages: Descriptive validation errors

3. ERROR HANDLING

   - Standardized ApiResponse interface
   - HTTP status codes: 400, 401, 403, 404, 500
   - Database error mapping
   - Detailed error messages for debugging
   - Silent failures for non-critical operations (e.g., image deletion)

4. AUTHENTICATION

   - Middleware: withAdminAuth()
   - Verifies: Supabase auth + admin role
   - Context: userId, email, IP, user agent
   - Future: Role-based access control (RBAC)

5. AUDIT LOGGING

   - Separate audit_logs table
   - All admin actions logged
   - Append-only for compliance
   - Queryable for analytics
   - Not blocking (logs asynchronously)

6. RATE LIMITING

   - In-memory store (development)
   - Redis support (production)
   - Per-user rate limit: 100 req/minute
   - 429 Too Many Requests response

7. FILE HANDLING

   - Supabase Storage (scalable, CDN)
   - Unique filenames with timestamp
   - Validation before upload
   - Auto-cleanup on delete

================================================================================
                        PERFORMANCE NOTES
================================================================================

Database Queries:
  - All list endpoints paginated (avoid N+1 queries)
  - Indexes on frequently filtered columns
  - Selective columns in some queries
  - Connection pooling recommended

File Uploads:
  - Stored in Supabase Storage (S3-compatible)
  - CDN-ready with public URLs
  - Async processing possible
  - Cleanup on delete

Rate Limiting:
  - In-memory for development
  - Scales to Redis for production
  - Per-user not per-IP
  - Configurable limits

Error Handling:
  - No blocking errors
  - Audit logs written asynchronously
  - Image upload failures don't block
  - Graceful degradation
