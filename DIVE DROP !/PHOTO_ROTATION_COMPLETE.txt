================================================================================
SCORING ALGORITHM
================================================================================

HOW PHOTOS ARE SELECTED:

Step 1: Fetch Approved Photos
  - From last 30 days
  - Only approved status
  - Per site

Step 2: Score Each Photo (0-100)
  - Recency (40%): 100 - days_old
  - Rating (30%): (rating / 5) * 100
  - Engagement (30%): min(100, comments*5 + views/10)

  Total: (recency * 0.4) + (rating * 0.3) + (engagement * 0.3)

Step 3: Select Winner
  1. Sort all photos by score (highest first)
  2. Take top 10
  3. Randomly pick 1 from top 10
  4. Use as hero image

EXAMPLE:
  Photo A: 99 (recency) * 0.4 + 90 (rating) * 0.3 + 55 (engagement) * 0.3
         = 39.6 + 27 + 16.5 = 83.1

  Photo B: 93 (recency) * 0.4 + 100 (rating) * 0.3 + 10 (engagement) * 0.3
         = 37.2 + 30 + 3 = 70.2

  Photo A wins (83.1 > 70.2)

================================================================================
TROUBLESHOOTING GUIDE
================================================================================

ISSUE: Cron job not running automatically
SOLUTION:
  1. Check vercel.json exists in root directory
  2. Verify deployment succeeded
  3. Go to Vercel dashboard > Settings > Crons
  4. Ensure endpoint is listed there
  5. Try manual trigger first
  6. Check Vercel cron logs for errors

ISSUE: No photos are being rotated
SOLUTION:
  1. Check pending photos: GET /api/admin/photos/approve
  2. Approve test photos if pending
  3. Ensure photos are < 30 days old
  4. Check /api/cron/photos/stats for errors
  5. Review database for site_photos records

ISSUE: Wrong photos being selected
SOLUTION:
  1. Review scoring algorithm in docs/PHOTO_ROTATION_SYSTEM.md
  2. Check scoringWeights sum to 1.0 in src/lib/cron/config.ts
  3. Enable verbose scoring: DEBUG_PHOTO_SCORING=true
  4. Test with testSelectionAlgorithm() helper
  5. Review individual photo scores in logs

ISSUE: High execution time
SOLUTION:
  1. Check Supabase database performance
  2. Verify indexes are created (they should be)
  3. Reduce topPhotosCount if very high
  4. Monitor with /api/cron/photos/stats
  5. Check query performance in Supabase dashboard
